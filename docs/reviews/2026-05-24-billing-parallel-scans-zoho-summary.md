# Краткий итог изменений

Дата: 2026-05-24T00:00:00Z

## Что изменилось

- **Биллинг**: в продукте появился более понятный billing shell. Он показывает текущий тариф, статус подписки, лимиты, использование и путь к апгрейду или управлению подпиской.
- **Параллельные сканы**: выбор сканов стал явным на уровне UI и backend-планирования. Пользователь видит доступные lanes, может выбрать нужные источники и сразу понять, какие из них доступны, в ожидании или временно недоступны.
- **Zoho CRM**: добавлена outbound-интеграция для активного workspace. VibeScan отправляет в Zoho только краткое состояние workspace: план, billing state, health сканов, дату последнего скана и число критических находок.

## Что важно

- Zoho-синк идет через отдельный backend flow с очередью, retry и reconciliation.
- Интеграция не отправляет в Zoho сырые findings, артефакты сканов или другие чувствительные детали.
- В Settings теперь можно подключить, проверить, пересинхронизировать и отключить Zoho без обращения к коду или API напрямую.

## Коротко

Это три связанные улучшения вокруг коммерческого и операционного контура продукта:

- биллинг стал читабельнее для пользователя;
- параллельные сканы стали понятнее и прозрачнее;
- Zoho получил безопасный outbound-синк по workspace summary.
